PROJECT 04 · INSPECTION REPORT

Amazon Clone

A responsive front-end clone of Amazon featuring a clean product layout, navbar, and hover interactions.

CLASSUI Replication
ENGINEHTML / CSS
SYSTEMSSemantic HTML
STATUSDEPLOYED
GitHub Live Demo
01

Overview

A UI replication project: rebuilding Amazon's product listing page — navbar, hero banner, product grid, and hover interactions — using only semantic HTML and modern CSS, with no framework and no backend. The goal was pixel-level layout accuracy, not a functioning store.

02

The Challenge

Amazon's real layout uses dense, irregular grids — different card sizes, overlapping promotional banners, and a navbar with several distinct interactive zones (search, account, cart, categories) packed into one bar. Reproducing that visual density accurately with plain CSS, without utility-class shortcuts or a component library, meant getting spacing and alignment exactly right by hand.

03

The Engineering

Built with semantic HTML elements (nav, section, article) rather than generic divs, so the structure carries meaning on its own, and CSS Grid/Flexbox handle the layout rather than fixed pixel positioning — which is part of why the result stays responsive across screen sizes despite the visual density of the original design.

04

Tech Stack

HTML5
CSS3
05

Key Features

  • Multi-zone navbar

    Search, account, and cart areas replicated as distinct interactive regions.

  • Dense product grid

    Variable-sized product cards laid out with CSS Grid.

  • Hover interactions

    Navbar and product card hover states matching the source design.

06

Results

The clone is deployed and live (linked above), reproducing Amazon's product-listing layout, navbar, and hover behavior with plain HTML/CSS — no component framework involved. It stands as a UI-fidelity and layout exercise rather than a functioning storefront.

07

Lessons

Cloning a real, high-density production UI by hand is a different skill from building an original layout from scratch — it forces close attention to spacing, alignment, and hover-state details that are easy to skip when there's no reference to hold yourself to. It was a useful exercise in reading someone else's design decisions and reproducing them precisely with plain CSS.
